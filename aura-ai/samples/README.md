# AURA AI Sample Outputs

This folder contains example outputs from the AURA AI pipeline for reference and testing purposes.

## Files

### `sample_output.json`
Complete API response from a successful `/aura/run` call when all signals are aligned and a trade signal is generated.

**Key fields:**
- `status`: "trade_signal" (also can be "no_trade" or "error")
- `confidence_score`: 85 (0-100, higher = more confident)
- `strategy_spec`: Backtestable trading specification
- `all_signals`: Full output from all 6 agents

### `sample_strategy_spec.json`
Just the strategy specification portion, useful for:
- Passing to backtesting engines (Freqtrade, etc.)
- Understanding the exact entry/exit conditions
- Extracting asset universe for trading

## Using These Samples

### For Testing
```python
import json
with open('samples/sample_output.json') as f:
    expected = json.load(f)
```

### For Documentation
Include these in your API docs to show clients what a live response looks like.

### For Backtesting
Extract the `strategy_spec` and feed it to your backtesting framework:
```python
spec = json.load(open('samples/sample_strategy_spec.json'))
# Pass to freqtrade, backtrader, zipline, etc.
```

## Real Response vs. Sample

The sample shows a **"trade_signal"** generated by Gemini API with 85% confidence.

In practice, you may see:
- `"no_trade"` — signals are mixed, not enough agreement → position_size = "0%"
- `generated_by_fallback: true` — Gemini API unavailable, using deterministic spec
- Lower confidence scores when fewer than 3/6 signals are aligned
